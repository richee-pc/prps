1. 미션 브리핑

정보 요원 여러분,

이번 6주간의 미션은 단순한 코딩 과제가 아닙니다. 이것은 여러분의 아이디어를 실제 작동하는 프로그램으로 구현하는 '창조'의 과정입니다.

성적을 넘어, 교실 밖의 실제 문제를 발견하고, 논리적으로 분석하며, 팀과 협력하여 해결하는 '진짜 실력'을 증명할 시간입니다. 이 과정에서 여러분은 막히고, 토론하고, 때로는 좌절하겠지만, 결국 문제를 해결해내는 '개발자'의 경험을 하게 될 것입니다.

여러분의 번뜩이는 아이디어로 세상을 "헉!" 하게 만들 준비가 되었습니까?
미션을 수락하십시오.

팀명, 주제 선정 이유, 핵심 알고리즘 설계도 (순서도, 의사코드 등), 수집/분석 자료, 최종 코드의 핵심 로직 설명, 시연 영상/스크린샷, 팀원별 역할 및 기여도.

[팀 제출]

4. 6주간의 여정 (The 6-Week Sprint)

시간 관리는 이 미션의 핵심입니다. 주차별 목표를 명확히 인지하고 완수하십시오.

1차시

[LAUNCH] 팀 빌딩 및 환경 구축

Mission: 승리를 위한 최고의 팀을 구성하고, 개발 환경을 완벽히 세팅한다.

  • Task 1: 팀 구성 및 프로젝트 주제 선정 (아이디어 구체화)
  • Task 2: '가상컴퓨터(VM)' 설치 (아래 '필수 자원' 매뉴얼 참고)
  • Task 3: 개발 환경 구축 (Anaconda, VSCode 설치 및 설정)
2차시

[PLAN] 자료 조사 및 알고리즘 설계

Mission: 아이디어를 '설계도'로 만든다.

Task: 국내/외 선행 자료 심층 조사, 프로젝트의 핵심 기능 정의, 알고리즘 구상 (순서도, 의사 코드 등 구체적인 설계도 작성)

3차시

[BUILD] 설계 및 프로토타이핑

Mission: 본격적인 코드 구현을 시작한다.

  • Task 1: (2차시 연장) 알고리즘 상세 설계 완료
  • Task 2: 핵심 기능(모듈)부터 코드 제작 시작 (프로토타입 빌드)
  • Task 3: 팀원 간 코드 공유 및 버전 관리 룰 정하기 (예: Git, Google Drive)
4차시

[DEVELOP] 코드 구현 및 고도화

Mission: 기능 확장 및 발견되는 오류를 수정한다.

  • Task 1: 팀원 간 코드 리뷰 및 통합 (중요!)
  • Task 2: 발견되는 버그 수정 및 예외 처리 로직 추가
  • Task 3: 발표 자료(Canva) 제작 시작 (알고리즘 설계 부분부터 채우기)
5차시

[FINALIZE] 최종 디버깅 및 발표 준비

Mission: 프로젝트를 '완성'하고 최종 리허설을 진행한다.

  • Task 1: 최종 코드 완성 및 가독성을 위한 주석(Comments) 작업
  • Task 2: 최종 발표 자료(Canva) 완성 (시연 영상/캡처 포함)
  • Task 3: 팀 내 최종 발표 리허설 (시간 배분, Q&A 대비)
6차시

[REPORT] 최종 발표 및 평가

Mission: 6주간의 여정을 공유하고 증명하라!

  • Task 1: 팀별 최종 프로젝트 발표 (발표 7분, Q&A 3분 엄수)
  • Task 2: (개별) 자기 평가 및 동료 평가 제출 (수업 시간 내 완료)

5. 필수 자원 (Essential Toolkit)

최고의 요원은 도구를 탓하지 않지만, 최고의 도구를 사용합니다.

[필독] 가상컴퓨터(VM) 설치 매뉴얼

*모든 개발은 이 가상 환경 내에서 진행하는 것을 원칙으로 합니다.

개발 환경: Anaconda (Python 배포판)

개발 환경: Visual Studio Code (편집기)

발표 자료 제작: Canva (캔바)

*팀 공유 기능을 적극 활용하세요.

6. 공식 평가 기준 (Official Rubric)

여러분의 프로젝트는 다음의 공식 평가 기준표(100점 만점)에 따라 평가됩니다. 각 항목의 세부 기준을 "헉" 소리 나게 꼼꼼히 확인하여, 최고의 성과를 거두시기 바랍니다. (총점 100점, 반영비율 40%)

평가 요소 만점 채점 기준 (수행 수준)
문제 발견 및
해결 방안 탐구		 20 [20점]

실생활에서 문제를 발견하여 주제를 선정하고 문제 해결을 위해 주제 선정 목적과 의도를 명확히 정의하며, 창의적이고 새로운 가치와 아이디어로 다양한 해결 방법을 제시함.
[15점]

실생활에서 문제를 발견하여 주제를 선정하였으나 기존 아이디어들과 거의 유사하며, 창의적이고 새로운 가치와 아이디어지만 한 가지 해결 방법만 제시함.
[10점]

실생활에서 문제를 발견하여 주제를 선정하였으나 프로그래밍으로 해결하기 어려우며, 문제 해결 방법을 찾았지만 창의적이고 새로운 가치와 아이디어가 아님.
[5점]

실생활에서 문제를 발견하여 주제를 선정하지 못하며, 문제 해결 방법을 제시하지 않음.
프로그램 개발 40 [40점]

문제해결에 충분한 소스코드를 LOC 20줄 이상으로 구성하고, 프로그래밍 실행 과정 중 발생하는 모든 오류를 수정하여 오류가 발견되지 않음.
[30점]

문제해결에 적절한 소스코드를 LOC 15~19줄로 구성하고, 프로그래밍 실행 과정에서 오류가 1개 발견됨.
[20점]

문제해결에 기본적인 소스코드를 LOC 10~14줄로 구성하고, 프로그래밍 실행 과정에서 오류가 2개 발견됨.
[10점]

문제해결을 위한 소스코드가 LOC 10줄 미만으로 부족하고, 프로그래밍 실행 과정에서 오류가 3개 이상 발견됨.
포트폴리오
작성 및 발표		 25 [25점]

작성한 프로그램 코드를 모두가 이해할 수 있게 정확하고 구체적으로 주석을 작성하여 소스코드를 구조화하여 명확하게 설명하고, 포트폴리오 발표자료 내용이 논리적이고 명료하며 자신감 있는 목소리로 발표함.
[20점]

작성한 프로그램 코드 주석을 정확하게 작성하여 소스코드를 적절히 설명하고, 포트폴리오 발표자료 내용은 논리적이고 명료하나 발표에 자신감이 다소 부족함.
[15점]

작성한 프로그램 코드 주석을 의도에 맞게 작성하여 기본적인 설명을 제공하고, 포트폴리오 발표자료 내용이 무난하고 발표 자신감도 보통 수준임.
[5점]

작성한 프로그램 코드 주석 작성에 어려움이 보이며 소스코드 설명이 부족하고, 포트폴리오 발표자료 내용이 논리적이지 않고 명확하지 않으며 발표도 요지를 파악할 수 없을 만큼 미흡함.
프로그램 공유 5 [5점]

작성한 프로그램을 클라우드를 통해 공유함.
[1점]

작성한 프로그램을 클라우드를 통해 공유하지 않음.
태도 및 참여도 10 [10점]

문제해결을 위해 적극적으로 의견을 나누고 다양한 정보를 공유하며 자신이 맡은 역할을 성실히 해냄.
[7점]

문제해결을 위해 의견을 나누고 정보를 공유하며 자신이 맡은 역할을 어느정도 해냄.
[4점]

문제해결을 위해 최소한의 의견과 정보만 공유하며 자신이 맡은 역할을 조금만 해냄.
[1H점]

문제해결을 위한 의견을 나누거나 정보를 공유하지 않고 자신이 맡은 역할을 해내지 못함.

최종 성취 수준 (A/B/C)

A 등급: 80점 이상

B 등급: 80점 미만 ~ 60점 이상

C 등급: 60점 미만

* 성취기준: C 프로그래밍 기초 개념을 이해하고, 프로그램 기획부터 설계, 구현, 테스트까지 전 과정을 효율적으로 완성할 수 있다.

7. 교사의 메시지 (A Message from HQ)

지난 10년간 수많은 학생들의 프로젝트를 봐왔습니다. 가장 빛나는 프로젝트는 기술적으로 가장 화려한 코드가 아니라, '왜' 이 문제를 풀어야 하는지에 대한 고민이 깊고, '함께' 벽을 넘기 위해 끝까지 노력한 팀의 것이었습니다.

6주간의 여정은 분명 험난할 것입니다. 막힐 때는 주저 말고 도움을 요청하십시오. 동료에게, 그리고 저에게. 여러분의 도전을 전폭적으로 지지합니다.
